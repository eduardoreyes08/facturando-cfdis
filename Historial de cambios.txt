==============================
+ Se agregó la funcionalidad para poder validar un CFD o CFDI

+ Se agregó la funcionalidad para generar un CFD o CFDI con los siguientes 
  complementos:
    . Divisas
    . Donatarias
    . Estado de cuenta bancario
    . Estado de cuenta combustible
    . Impuestos locales
    . Prestadores de servicios de CFD
    . Timbre Fiscal Digital

+ Se agregó el método IsValidToSign a la clase ElectronicCertificate, se usa 
  cuando se desea verificar si un certificado se puede usar para firmar un 
  CFD/CFDI. Es un método de clase que puede ser usado cuando se guarda el 
  certificado en la base de datos, antes de subirlo se ejecuta y con esto se 
  puede eliminar ciertas validaciones en el proceso de generación de un 
  documento. Las validaciones que realiza este método son:

    . Si el certificado es válido contra una autoridad certificadora
    . Si el certificado es un certificado de sello
    . EL RFC del emisor es igual al RFC registrado en el certificado. 

  Al momento de generar el CFD/CFDI se pueden eliminar estas validaciones.
  

+ Se agregó la propiedad FingerPrintSAT a la clase ElectronicDocument, esta 
  permite obtener la cadena original que uso el PAC para generar el sello del 
  SAT. Esta cadena es usada en la impresión de un CFDI. Para obtener esta 
  cadena es necesario:

    . Que la versión del documento sea 3.0 
    . Que el documento sea leído con el método LoadFromFile y que este 
      contenga el complemento Timbre Fiscal. 
    . Que durante la generación le sea asignado al documento el complemento 
      Timbre Fiscal.

+ Se agregó el método Diggest a la clase ElectronicDocument, este permite 
  obtener la digestión de las cadenas originales; para esto es necesario pasar
  un parámetro indicando cual digestión es la que se desea obtener.

+ Se agregó el método Extract a la clase FolioAutorizadoList; este método 
  rellena una lista con objetos de la clase FolioDescription que contienen la 
  información detallada de los folios que han sido autorizados por el SAT. 
  Ha sido creado para que la información del archivo FoliosCFD.txt sea subido 
  a una base de datos y no se use dicho archivo en el proceso de validación. 
  Este método podría ser útil en los siguientes casos:

    . Aplicaciones Web, ya que no se cargaría el archivo de folios a memoria.
    . Aplicaciones de escritorio en las que no se quiera depender de un archivo
      externo.

  Su uso está ligado al evento FolioAutorizadoList.OnValidating

+ Se agregó el método Extract a la clase CertificateRevocationList; este método
  rellena una lista con objetos de la clase CertificateDescriptionque contienen 
  la información detallada de los certificados de sello que han sido 
  autorizados 
  por el SAT. Ha sido creado para que la información del archivo CSD.txt sea 
  subido a una base de datos y no se use dicho archivo en el proceso de 
  validación. Este método podría ser útil en los siguientes casos:

    . Aplicaciones Web, ya que no se cargaría el archivo de certificados de 
      sello a memoria
    . Aplicaciones de escritorio en las que no se quiera depender de un archivo
      externo.

+ Se agregó un ejemplo (06. Archivos SAT) para demostrar el manejo de los 
  archivos del SAT en una base de datos.

  agregar el timbre al CFDI.
